# SEO Plan: Bewaard voor Baby

> Datum: 26 juni 2026
> Doel: organisch gevonden worden op de juiste zoektermen rondom babyboeken, kraamcadeaus en het eerste levensjaar van een baby.

---

## 1. Huidige situatie

Bewaard voor Baby staat op:
- `/voor-baby` — landingspagina (al geïndexeerd, maar geen backlinks, geen content)
- `/voor-baby/hoe-het-werkt` — informatiepagina
- `/voor-baby/over-ons` — over ons
- `/voor-baby/onboarding` — hidden (ingelogd)
- `/voor-baby/dashboard` — hidden (ingelogd)

**Wat er ontbreekt:**
- Geen kennisbank-artikelen of blog over baby-gerelateerde onderwerpen
- Geen interne links vanuit de bestaande kennisbank naar voor-baby
- Geen backlinks / externe signalen
- Geen gestructureerde content (geen FAQ schema, geen how-to schema)
- Het subdomein (of subfolder) heeft geen eigen content-marketing funnel
- `BABY_GIFT` checkout-pagina is niet geoptimaliseerd voor zoekmachines (geen content)

### 3.3 Onze unieke voordelen (USP's voor SEO)

1. **Geen app** — werkt via e-mail (SEO-friendly: "geen app nodig", "werkt via email")
2. **Eenmalige betaling** — geen abonnement (SEO: "eenmalig €59", "geen abonnement")
3. **Grootouder-updates** — uniek, geen concurrent heeft dit (SEO: "opa en oma automatisch updaten")
4. **AI-gedreven vragen** — uniek in NL (SEO: "AI vragen babyboek", "automatische vragen eerste jaar")
5. **Rol-bewust** — moeder, partner, samen (SEO: "partner schrijft mee")
6. **Fotoboek-voucher** na elk jaar (SEO: "gratis fotoboek baby")
7. **Privacy** — Europese servers, AVG (SEO: "veilig babyfoto's delen")
8. **Van BewaardVoorJou** — bestaand merk (autoriteit overdragen)

### 4.3 Landingspagina optimalisatie

**Huidige pagina's:**
- `/voor-baby` — goed, maar mist gestructureerde data

**Wat toevoegen aan `/voor-baby`:**

- HowTo schema (de 7 stappen)
- Product schema (BABY_GIFT als product met prijs €59)

- Meer content onder de fold: een sectie met "veelgestelde vragen" direct op de landingspagina

### 4.4 Interne link structuur

Momenteel geen links naar voor-baby vanuit de bestaande kennisbank.

**Cruciale interne links om te leggen:**

```
/kennisbank/digitale-nalatenschap   →  /voor-baby (zijdelings: "bewaar ook het eerste jaar")
/kennisbank/levensverhaal-vastleggen →  /voor-baby (gerelateerd: herinneringen bewaren)
/kennisbank/...[elk baby-artikel]   →  CTA naar /voor-baby of /checkout?package=BABY_GIFT
Blog artikelen                       →  interne links naar kennisbank baby-artikelen
```

**Omgekeerd:**
```
/voor-baby           →  /kennisbank/babyboek-eerste-jaar  (als extra leestip)
/voor-baby/hoe-het-werkt  →  /kennisbank/babyherinneringen-bewaren
```

**Breadcrumbs:**
```
Home → Voor Baby → ...
```

### 4.5 CTA optimalisatie

Niet alleen onderaan, maar ook middenin (tussen paragrafen) een subtiele CTA.

Tekst: "Geef een babyboek cadeau → €59 eenmalig" of "Bewaar het eerste jaar van jouw baby →"

## 5. Technische SEO

### 5.1 Sitemap & indexatie

- `/voor-baby` → ✅ in sitemap
- `/voor-baby/hoe-het-werkt` → ✅ in sitemap
- `/voor-baby/over-ons` → ✅ in sitemap
- `/voor-baby/onboarding` → ⛔ noindex (ingelogd)
- `/voor-baby/dashboard` → ⛔ noindex (ingelogd)
- `/checkout?package=BABY_GIFT` → ⛔ noindex (checkout flow)
- Nieuwe kennisbank artikelen → ✅ index, voeg direct toe aan sitemap

### 5.2 Schema.org markup

Op `/voor-baby`:
```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Bewaard voor Baby",
  "description": "Het digitale babyboek als kraamcadeau. Een jaar lang herinneringen bewaren met AI-begeleide vragen.",
  "offers": {
    "@type": "Offer",
    "price": "59",
    "priceCurrency": "EUR",
    "priceValidUntil": "2027-06-26",
    "availability": "https://schema.org/InStock"
  },
  "category": "Digital Baby Book"
}
```

Op `/voor-baby/hoe-het-werkt`:
```json
{
  "@context": "https://schema.org",
  "@type": "HowTo",
  "name": "Hoe werkt Bewaard voor Baby",
  "step": [...]
}
```

Op kennisbank artikelen:
```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "...",
  "description": "..."
}
```

### 5.3 OpenGraph / social

✅ Al aanwezig in de metadata van elke voor-baby pagina.
- Wel de **hero afbeelding** optimaliseren: `/images/baby-hero.png` moet minimaal 1200x630px zijn voor social cards
- Overweeg een aparte OG-afbeelding per artikel

### 5.4 Core Web Vitals

- De pagina's gebruiken veel client-side interactie (thema-kiezer, FAQ accordeon)
- Overweeg: FAQ markup aan de server-kant zetten (Next.js metadata) zodat Google het direct ziet zonder JS
- Baby-hero afbeelding heeft al `priority` en `sizes` ✅
- Overweeg afbeeldingen voor de 14 thema's en features te comprimeren

## 7. Conversie optimalisatie

### 7.1 Belangrijkste pagina's per zoekintentie

| Keyword type | Bestemmingspagina |
|---|---|
| Transactie (kopen) | `/checkout?package=BABY_GIFT` of `/voor-baby` |
| Informatie (vergelijken) | Specifiek kennisbank artikel |
| Informatie (probleem) | Kennisbank artikel → interne link → `/voor-baby` |
| Navigatie (merk) | `/voor-baby` |

### 7.2 CTA flow optimaliseren

Op elk kennisbank artikel:
- **Bovenin**: directe CTA ("Geef als kraamcadeau")
- **Midden**: contextuele CTA (bijv. na bespreking van mijlpalen: "Wil je mijlpalen bijhouden zonder gedoe? →")
- **Onder**: hoof CTA met prijs

**Let op:** De checkout pagina (`?package=BABY_GIFT`) heeft zelf weinig SEO-content. Overweeg een simpele product-pagina met content voor SEO, die doorlinkt naar checkout.
